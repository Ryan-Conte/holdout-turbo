# HOLDOUT — agent guide

Top-down multiplayer survival extraction shooter (DayZ / Rust / ARC Raiders loop, Zero Sievert presentation). Turborepo + npm workspaces.

## Layout

- `packages/shared` — recipes, trader stock, tiles, socket protocol types. **Single source of truth**; both apps import `@holdout/shared` (compiled to `dist/`, run `npm run build -w @holdout/shared` after edits). Item definitions live in `src/items.ts` — a category-builder registry (`weapon()/tool()/food()/placeable()` etc.); add an item by extending the `ItemId` union + one builder call. UI verbs come from item flags (`food/drink/heal/place/armor/weapon`) via `useVerb()`, so new items need no client changes.
- `apps/api` — NestJS + socket.io **authoritative** game server (port 3001). All simulation server-side: movement, combat, harvesting, loot, trading, hideouts. Clients only send intents.
- `apps/web` — Next.js 15 (App Router, React 19). Better Auth (email/password), canvas game client, and the admin-only game engine at `/admin/{slug}` (`/editor` redirects).
- `tools/gen-sprites.mjs` — generates placeholder pixel-art sprite sheets into `apps/web/public/sprites/*.png`. Run `node tools/gen-sprites.mjs`. Replace PNGs with real art later; keep grid layout documented in `docs/ART.md`.

## Key flows

- **Database**: Prisma 6 (schema at `prisma/schema.prisma`, client generated to root node_modules, shared by both apps). Change schema → `npx prisma db push` + `npx prisma generate` at root. Runtime URL from env `DATABASE_URL`; `DIRECT_DATABASE_URL` (non-pooler host) for DDL. Do NOT upgrade to Prisma 7 without migrating to its driver-adapter config.
- **Auth/profile**: Better Auth on the Prisma adapter owns `user/session/account/verification`. Email/password plus **Steam login** (`/api/auth/steam` → OpenID 2.0 → `/api/auth/steam/callback` bridges the SteamID into a shadow Better Auth account: email `steam-<id>@steam.holdout.local`, password HMAC-derived from `BETTER_AUTH_SECRET`; `STEAM_API_KEY` env optional for persona names). The game socket uses a short-lived JWT from `POST /api/game-token` (web, session-gated) verified by the API (`JWT_SECRET` shared via env). Game profile data lives in `profiles.data` jsonb, including inventory/equipment/progression and layered `appearance`. The landing briefing requires a saved appearance before first deploy; `/api/profile/appearance` is session-gated and legacy `look` values migrate lazily.
- **Stamina**: authoritative in `GameService`. Sprint can drain through zero; zero latches `staminaExhausted`, waits `STAMINA_EXHAUSTED_REGEN_DELAY_MS`, and keeps sprint locked until `STAMINA_EXHAUSTED_RECOVERY`. Ordinary exertion uses the shorter regen delay. Never make sprint eligibility client-authoritative.
- **Combat**: magazine + reload per weapon (`WeaponStats.magSize/reloadMs`, server-tracked `p.mags`), armor mitigation, weapon mods in the `mod` equipment slot (red dot = spread, suppressor = aggro noise; bows have `noise` overrides), skill bonuses (`skillLevel` from xp in profiles). Disconnecting in the open leaves a killable body for 60s (`loggedOutAt`) — no combat logging.
- **Economy/progression**: ore-veined rocks (`Tile.CopperOre/IronOre`, `COPPER_CHANCE/IRON_CHANCE` in mapgen) yield stone + ore (`ORE_YIELD`); furnace SMELTS only (`cat:'smelt'`), anvil forges weapons/ammo/attachments (`cat:'forge'`), workbench makes tools/gear/kits. Station recipes are hidden client-side unless standing at the station. High-loot zones (`PoiSnap.hot`, editor `poi_hotzone`, airport is hot by default) upgrade chests to the `rare` loot table (dmr/lmg/prototype, attachments, valuables like gold bars/diamonds/artifacts that exist to be sold/shown off). Traders have tiers (`TRADER_TIER_STOCK`): outpost = 1, black-market dealer (`trader_black`, unsafe) = 2 and pays big for valuables.
- **Server browser & admins**: `game_servers` table → `/api/servers` (public, auto-seeds Local) + `/api/admin/servers`; players pick a server on the login page (localStorage `holdout_server_url`), game client connects there. Admins are a DB flag (`profiles.admin`) via `requireAdmin()`; `ADMIN_EMAILS` env only bootstraps the flag on first check.
- **Deployment**: `apps/web/vercel.json` targets Vercel with `apps/web` as the project root. Root `render.yaml` provisions separate regional API worlds; each uses a distinct `PUBLIC_SERVER_URL` and self-registers in `game_servers`. All deployments share Postgres and `JWT_SECRET`, but API processes never share simulation memory and must not be autoscaled behind one URL. See `docs/DEPLOYMENT.md`.
- **Hideouts ("home base")**: per-user grass-field instance ringed by impassable cliffs (`HIDEOUT_W×HIDEOUT_H`, bed + starter stash). Building is craft-then-place: hold a `kit_*`, press `R` to rotate its DB-art ghost, then click; `c:build {slot,tx,ty,rotation}` persists the orientation. Doors swing open near players, temporarily allowing enemies, bullets and sight through, then close when clear. Floors remain foundations, chests retain storage, and world builds wear out while camp builds persist.
- **Quests**: `quests` table (with `requiresId` unlock chain + `tier`), CRUD via `/api/admin/quests` (editor QUESTS tab), hot-reloaded by the API every 60s; `tools/seed-quests.mjs` seeds the test tree. A quest only appears (and only ticks kills) once its prerequisite is claimed, and only at a trader of its tier. Kill quests tick on enemy death; fetch quests check inventory at claim.
- **Chat**: `c:chat` allowed only in safe zones/hideouts, broadcast per instance, rendered as overhead bubbles + log.
- **Instances**: the server runs named instances — `world` plus on-demand `h:<userId>` hideouts. Sockets join a room per instance; state broadcasts are per-instance. Changing instance re-sends `s:init`.
- **Extraction**: world maps have extraction beacons (`inst.extracts` — authored `extract` objects or `placeExtracts` edge fallback). `E` nearby starts a 5s hold-still action (`EXTRACT_TIME_MS`); success switches you to your hideout with loot intact. Beacons refuse inside safe zones, and `hideoutEnter` no longer warps you to your own camp — extraction is the only way home (friends' camps still visitable from safe zones). Death still drops everything.
- **Line of sight**: `losClear()` raycasts `BLOCKS_BULLET` tiles: enemies can't acquire targets through walls or forest, and `broadcast()` sends per-viewer snapshots in world instances — entities without LOS (beyond `SENSE_RANGE`) are omitted entirely, so wallhacks see nothing. Enemies have sight memory (`lastSeenAt/X/Y`, `LOS_MEMORY_MS`): they hunt your last-seen spot, only attack what they can see, investigate gunshot origins, and forget you ~3s after you break LOS — hiding works. The client mirrors vision with an LOS fog overlay (`WorldView.fog`, renderer `losTo`). (Survivor NPC bots were removed by request.)
- **Stations on floors**: placing a piece on flooring records the floor in `inst.unders` (sent via `WorldInit.unders` + `TileUpdate.under`); the renderer draws it beneath the station and demolish/destroy/decay restores the floor, not grass. Near-station flags push whenever `lastStationMask` changes, so `nearWorkbench/…` are never stale. Node respawns re-roll: rock can come back ore-veined; a mined vein usually reverts to plain rock (no ore camping). Friends' camps are visitable from your own base or safe zones (leaving returns you home / to the safe zone).
- **Day/night**: global clock (`DAY_LENGTH_MS`, `isNight()`). At night zombies get `NIGHT_SPEED_MULT`/`NIGHT_AGGRO_MULT`, wolves aggro wider; client renders deeper darkness, dusk/dawn tint, torch/firepit glow.
- **Engine visuals/terrain**: `sprites` stores multi-frame RGBA assets; `animations` maps `player` and `mob:<id>` state clips to frame order/timing. Published visuals are sent in `WorldInit` and hot-pushed with `s:visuals`; sheet art remains fallback. Authored maps may carry `elevations` (0-3): one-level grades are hills, jumps over one level block movement/LOS/projectiles as cliffs.
- **Safe zones**: POIs with `safe: true` (trader outposts). No damage in/out, enemies don't aggro, shooting disabled. Hideouts are always safe, and resting there heals (`HOME_REST_HP_PER_S`).
- **Game engine**: `/admin/{slug}` uses `game_content` draft/published documents. Mobs, recipes, loot and traders hot-reload every 10s; maps publish separately and load when the world is empty. See `docs/ENGINE.md`.
- **Maps**: server loads the active non-draft row from `maps` or falls back to procedural generation (`apps/api/src/game/mapgen.ts`). Authored maps can attach published resource IDs to tree/rock/ore tiles; those definitions control health, drops, respawn, sprite and sound.
- **Engine audio/animation**: published `sounds` are Web Audio presets referenced by action routes, mobs, resources and animation keyframes. Animation keyframes can override duration and carry event/sound metadata.
- **Dynamic world blocks**: `AuthoredMap.blocks` maps tile indices to published block IDs. The block catalog includes authored-only geometry and all player-placeable build pieces. Definitions reference pixel assets directly and control movement/enemy/bullet/sight collision, health, drops, sounds and optional `playerPlacement` kit/build metadata; terrain remains underneath. Player placement emits/restores block IDs while shared `BUILDABLES` tiles remain a station/foundation/persistence compatibility cache.

## Conventions

- Socket event names only via `EV` from shared — never string literals.
- Never trust client payloads: clamp numbers, bounds-check slots, rate-limit (see `apps/api/src/game/rate-limit.ts` and `docs/ANTICHEAT.md`).
- Env: `apps/api/.env`, `apps/web/.env.local` (gitignored). `JWT_SECRET` must match across both.
- `npm run dev` at root runs everything. `npx turbo run typecheck` before calling work done.

# HOLDOUT â€” agent guide

Top-down multiplayer survival extraction shooter (DayZ / Rust / ARC Raiders loop, Zero Sievert presentation). Turborepo + npm workspaces.

## Layout

- `packages/shared` â€” recipes, trader stock, tiles, socket protocol types and outage fallbacks. Both apps import `@holdout/shared` (compiled to `dist/`, run `npm run build -w @holdout/shared` after edits). Checked-in item definitions live in `src/items.ts`; published `game_content.items` can override or extend them at runtime. `WorldInit.gameplay`/`s:gameplay` keep clients on the server's sanitized item/recipe revision. UI verbs come from item flags via `useVerb()`.
- `apps/api` â€” NestJS + socket.io **authoritative** game server (port 3001). All simulation server-side: movement, combat, harvesting, loot, trading, hideouts. Clients only send intents.
- `apps/web` â€” Next.js 15 (App Router, React 19). Better Auth (email/password), canvas game client, and the admin-only game engine at `/admin/{slug}` (`/editor` redirects).
- `tools/gen-sprites.mjs` â€” generates placeholder pixel-art sprite sheets into `apps/web/public/sprites/*.png`. Run `node tools/gen-sprites.mjs`. Replace PNGs with real art later; keep grid layout documented in `docs/ART.md`.

## Key flows

- **Database**: Prisma 6 (schema at `prisma/schema.prisma`, client generated to root node_modules, shared by both apps). Change schema â†’ `npx prisma db push` + `npx prisma generate` at root. Runtime URL from env `DATABASE_URL`; `DIRECT_DATABASE_URL` (non-pooler host) for DDL. Do NOT upgrade to Prisma 7 without migrating to its driver-adapter config.
- **Auth/profile**: Better Auth on the Prisma adapter owns `user/session/account/verification`. Email/password plus **Steam login** (`/api/auth/steam` â†’ OpenID 2.0 â†’ `/api/auth/steam/callback` bridges the SteamID into a shadow Better Auth account: email `steam-<id>@steam.holdout.local`, password HMAC-derived from `BETTER_AUTH_SECRET`; `STEAM_API_KEY` env optional for persona names). The game socket uses a short-lived JWT from `POST /api/game-token` (web, session-gated) verified by the API (`JWT_SECRET` shared via env). Game profile data lives in `profiles.data` jsonb, including inventory/equipment/progression and layered `appearance`. The landing briefing requires a saved appearance before first deploy and renders that current survivor in the deployment portrait; `/api/profile/appearance` is session-gated and legacy `look` values migrate lazily.
- **Stamina**: authoritative in `GameService`. Sprint can drain through zero; zero latches `staminaExhausted`, waits `STAMINA_EXHAUSTED_REGEN_DELAY_MS`, and keeps sprint locked until `STAMINA_EXHAUSTED_RECOVERY`. Ordinary exertion uses the shorter regen delay. Never make sprint eligibility client-authoritative.
- **Combat**: magazine + reload per weapon (`WeaponStats.magSize/reloadMs`, server-tracked `p.mags`), armor mitigation, weapon mods in the `mod` equipment slot (red dot = spread, suppressor = aggro noise; bows have `noise` overrides), skill bonuses (`skillLevel` from xp in profiles). Disconnecting in the open leaves a killable body for 60s (`loggedOutAt`); expiry kills it and drops carried gear, so there is no free logout extraction.
- **Economy/progression**: ore-veined rocks (`Tile.CopperOre/IronOre`, `COPPER_CHANCE/IRON_CHANCE` in mapgen) yield stone + ore (`ORE_YIELD`); furnace SMELTS only (`cat:'smelt'`), anvil forges weapons/ammo/attachments (`cat:'forge'`), workbench makes tools/gear/kits. Firepits and furnaces store server-owned wood heat (`STATION_FUEL_*`); one wood adds four charges and every cook/smelt consumes one. Home fuel persists in hideout objects. Station recipes are hidden client-side unless standing at the station. High-loot zones upgrade chests to the `rare` table; tier-2 traders pay heavily for valuables.
- **Server browser & admins**: `game_servers` table â†’ `/api/servers` (public, auto-seeds Local) + `/api/admin/servers`; players pick a server on the login page (localStorage `holdout_server_url`), game client connects there. Admins are a DB flag (`profiles.admin`) via `requireAdmin()`; `ADMIN_EMAILS` env only bootstraps the flag on first check.
- **Deployment**: `apps/web/vercel.json` targets Vercel with `apps/web` as the project root. Root `render.yaml` provisions separate single-instance regional API worlds; each self-registers in `game_servers` from Render's external URL (`PUBLIC_SERVER_URL` is an optional override). All deployments share Postgres and `JWT_SECRET`, but API processes never share simulation memory and must not be autoscaled behind one URL. An isolated draft server uses `CONTENT_CHANNEL=staging`, stays unregistered by default, and is linked from the engine with `NEXT_PUBLIC_STAGING_GAME_URL`. See `docs/DEPLOYMENT.md`.
- **Hideouts ("home base")**: per-user grass-field instance ringed by impassable cliffs (`HIDEOUT_WĂ—HIDEOUT_H`, bed + starter stash). Building is craft-then-place: hold a `kit_*`, press `R` to rotate its DB-art ghost, then click; `c:build {slot,tx,ty,rotation}` persists the orientation. Doors swing open near players, temporarily allowing enemies, bullets and sight through, then close when clear. Floors remain foundations, chests retain storage, and world builds wear out while camp builds persist.
- **Quests**: `quests` table (with `requiresId` unlock chain + `tier`), CRUD via `/api/admin/quests` (editor QUESTS tab), hot-reloaded by the API every 60s; `tools/seed-quests.mjs` seeds the test tree. A job unlocks (and only ticks kills) after its prerequisite is claimed, then remains visible in the HUD tracker; rewards must still be claimed at a trader of its tier. Fetch jobs check inventory at claim.
- **Chat**: `c:chat` works anywhere, broadcasts within the sender's current instance, and renders as overhead bubbles + log. Guests remain chat-disabled.
- **Instances**: the server runs named instances â€” `world`, on-demand `h:<userId>` personal hideouts, and `clan:<id>` shared holdouts. Sockets join a room per instance; state broadcasts are per-instance. Changing instance re-sends `s:init`.
- **Clans**: `clans` + `clan_members` + `clan_invites` enforce one clan per survivor and Owner/Officer/Member ranks. `/api/clans` owns management. `clan:<id>` is a 42Ă—30 shared safe hideout persisted in `clans.hideout`; all members build/use storage, Owner/Officers demolish. A renewable `clan_hideout_leases` row prevents cross-server last-writer-wins corruption. `StateSnap.mapPlayers` contains server-authorized friend/clan tactical positions only.
- **Extraction**: world maps have extraction beacons (`inst.extracts` â€” authored `extract` objects or `placeExtracts` edge fallback). `E` nearby starts a 5s hold-still action (`EXTRACT_TIME_MS`); success switches you to your hideout with loot intact. Beacons refuse inside safe zones, and `hideoutEnter` no longer warps you to your own camp â€” extraction is the only way home (friends' camps still visitable from safe zones). Death still drops everything.
- **Line of sight**: `losClear()` raycasts blocking tiles/content. `broadcast()` sends bounded per-viewer world snapshots: players, enemies, projectiles, containers and ground loot outside range/LOS are omitted, while population remains aggregate. Enemies hunt last-seen positions and investigate gunshots. The client mirrors vision with an LOS fog overlay. Optional authoritative survivor bots are controlled by published settings and use the same movement/combat/economy rules.
- **Stations on floors**: placing a piece on flooring records the floor in `inst.unders`; demolish/destroy/decay restores it. Published player-block definitions and legacy tiles both resolve station behavior. Near-station flags push whenever `lastStationMask` changes, and firepit/furnace fuel is keyed by station tile and persisted for hideouts. Node damage, cooldowns and rerolled variants persist per server/map in `game_world_state`; regrowth weights tree variants (common/ironwood) and rerolls rock-family ore. Friends' camps are visitable from home or safe zones.
- **Day/night**: global clock (`DAY_LENGTH_MS`, `isNight()`). At night zombies get `NIGHT_SPEED_MULT`/`NIGHT_AGGRO_MULT`, wolves aggro wider. Client composites an offscreen darkness mask: equipped items with `lightRadius` reveal portable light, torch posts reveal the widest static area, and firepits illuminate only while server-replicated `stationFuel` is positive.
- **Engine visuals/terrain**: `sprites` stores multi-frame RGBA assets; `animations` maps `player` and `mob:<id>` state clips to frame order/timing. Published visuals are sent in `WorldInit` and hot-pushed with `s:visuals`; sheet art remains fallback. Authored maps may carry `elevations` (0-3): one-level grades are hills, jumps over one level block movement/LOS/projectiles as cliffs.
- **Safe zones**: POIs with `safe: true` (trader outposts). No damage in/out, enemies don't aggro, shooting disabled. Hideouts are always safe, and resting there heals (`HOME_REST_HP_PER_S`).
- **Game engine**: `/admin/{slug}` uses `game_content` heads plus immutable `game_content_revisions`. The versioned runtime item/recipe catalog is sent in `WorldInit` and `s:gameplay`; the global admin safety bar restores content/map history and launches a configured isolated staging raid. Maps load when the world is empty. See `docs/ENGINE.md`.
- **Maps**: server loads the active non-draft row from `maps` or falls back to procedural generation (`apps/api/src/game/mapgen.ts`). Authored maps can attach published resource IDs to tree/rock/ore tiles; those definitions control health, drops, respawn, sprite and sound.
- **Engine audio/animation**: published `sounds` are Web Audio presets referenced by action routes, mobs, resources and animation keyframes. Animation keyframes can override duration and carry event/sound metadata.
- **Dynamic world blocks**: `AuthoredMap.blocks` maps tile indices to published block IDs. The block catalog includes authored-only geometry and all player-placeable build pieces. Definitions reference pixel assets directly and control movement/enemy/bullet/sight collision, health, drops, sounds and optional `playerPlacement` kit/build metadata; terrain remains underneath. Player placement emits/restores block IDs while shared `BUILDABLES` tiles remain a station/foundation/persistence compatibility cache.

## Conventions

- Socket event names only via `EV` from shared â€” never string literals.
- Never trust client payloads: clamp numbers, bounds-check slots, rate-limit (see `apps/api/src/game/rate-limit.ts` and `docs/ANTICHEAT.md`).
- Env: `apps/api/.env`, `apps/web/.env.local` (gitignored). `JWT_SECRET` must match across both.
- `npm run dev` at root runs everything. `npm run check` must pass; it includes workspace typechecks, production builds and deterministic gameplay tests.

# HOLDOUT â€” agent guide

Top-down multiplayer survival extraction shooter (DayZ / Rust / ARC Raiders loop, Zero Sievert presentation). Turborepo + npm workspaces.

## Layout

- `packages/shared` â€” recipes, trader stock, tiles, socket protocol types and outage fallbacks. Both apps import `@holdout/shared` (compiled to `dist/`, run `npm run build -w @holdout/shared` after edits). Checked-in item definitions live in `src/items.ts`; published `game_content.items` can override or extend them at runtime. `WorldInit.gameplay`/`s:gameplay` keep clients on the server's sanitized item/recipe revision. UI verbs come from item flags via `useVerb()`.
- `apps/api` â€” NestJS + socket.io **authoritative** game server (port 3001). All simulation server-side: movement, combat, harvesting, loot, trading, hideouts. Clients only send intents.
- `apps/web` â€” Next.js 15 (App Router, React 19). Better Auth (email/password), canvas game client, and the admin-only game engine at `/admin/{slug}` (`/editor` redirects).
- `tools/gen-sprites.mjs` â€” generates placeholder pixel-art sprite sheets into `apps/web/public/sprites/*.png`. Run `node tools/gen-sprites.mjs`. Replace PNGs with real art later; keep grid layout documented in `docs/ART.md`.

## Key flows

- **Database**: Prisma 6 (schema at `prisma/schema.prisma`, client generated to root node_modules, shared by both apps). Change schema â†’ `npx prisma db push` + `npx prisma generate` at root. Runtime URL from env `DATABASE_URL`; `DIRECT_DATABASE_URL` (non-pooler host) for DDL. Do NOT upgrade to Prisma 7 without migrating to its driver-adapter config.
- **Auth/profile**: Better Auth on the Prisma adapter owns `user/session/account/verification`. Email/password plus **Steam login** (`/api/auth/steam` â†’ OpenID 2.0 â†’ `/api/auth/steam/callback` bridges the SteamID into a shadow Better Auth account: email `steam-<id>@steam.holdout.local`, password HMAC-derived from `BETTER_AUTH_SECRET`; `STEAM_API_KEY` env optional for persona names). The game socket uses a short-lived JWT from `POST /api/game-token` (Better Auth session or signed guest cookie) verified by the API (`JWT_SECRET` shared via env). Game profile data lives in `profiles.data` jsonb, including inventory/equipment/progression and layered `appearance`. The landing briefing requires a saved appearance before first deploy and renders that current survivor in the deployment portrait; `/api/profile/appearance` is session-gated and legacy `look` values migrate lazily.
- **Guests**: logged-out visitors may request a signed, six-hour HTTP-only guest identity from `POST /api/game-token` and enter the main world immediately. Guest JWTs require a strict `guest:<uuid>` subject/callsign shape. Guests are in-memory only, start with empty hands and zero credits, consume normal relay capacity, retain the 60-second combat-log body, and never acquire profile leases or write profiles/hideouts. Chat, friends, clans, hideout travel and extraction are server-blocked; death redeploys them into the world. Admins see a GUEST marker and relay-local guest sanctions.
- **Stamina**: authoritative in `GameService`. Sprint can drain through zero; zero latches `staminaExhausted`, waits `STAMINA_EXHAUSTED_REGEN_DELAY_MS`, blocks combat attacks, applies the same 45% movement multiplier as overweight (without stacking), and keeps sprint locked until `STAMINA_EXHAUSTED_RECOVERY`. Ordinary exertion uses the shorter regen delay. Consumables remain usable while exhausted. Never make sprint or attack eligibility client-authoritative.
- **Combat**: magazine + reload per weapon (`WeaponStats.magSize/reloadMs`, server-tracked `p.mags`), armor mitigation, weapon mods in the `mod` equipment slot (red dot = spread, suppressor = aggro noise; bows have `noise` overrides), skill bonuses (`skillLevel` from xp in profiles). Disconnecting in the open leaves a killable body for 60s (`loggedOutAt`); expiry kills it and drops carried gear, so there is no free logout extraction.
- **Economy/progression**: ore-veined rocks (`Tile.CopperOre/IronOre`, `COPPER_CHANCE/IRON_CHANCE` in mapgen) yield stone + ore (`ORE_YIELD`); furnace SMELTS only (`cat:'smelt'`), anvil forges weapons/ammo/attachments (`cat:'forge'`), workbench makes tools/gear/kits. Firepits and furnaces store server-owned wood heat (`STATION_FUEL_*`); one wood adds four charges and every cook/smelt consumes one. Home fuel persists in hideout objects. Station recipes are hidden client-side unless standing at the station. High-loot zones (`PoiSnap.hot`, editor `poi_hotzone`, airport is hot by default) upgrade chests to the `rare` loot table. Traders have tiers (`TRADER_TIER_STOCK`): outpost = 1, black-market dealer (`trader_black`, unsafe) = 2 and pays big for valuables.
- **Server browser & admins**: `game_servers` table â†’ `/api/servers` (public, auto-seeds Local) + `/api/admin/servers`; registered and guest players pick a server on the login page (localStorage `holdout_server_url`), and the game client connects to that relay. Admins are a DB flag (`profiles.admin`) via `requireAdmin()`; `ADMIN_EMAILS` env only bootstraps the flag on first check. Verified admins receive a cyan world/name/chat tag and an in-game `F10` world-control console. Every privileged socket action rechecks the DB role and is bounded/rate-limited: issue active-catalog items, go to/bring/send-home/restore players, teleport, protection/sight, relay announcements, kick, timed mute/ban, and sanction removal. Mutes/bans persist on `profiles`, bans are checked at token issue and socket admission, admins cannot sanction other admins, and actions write structured `admin_action` telemetry.
- **Deployment**: `apps/web/vercel.json` targets Vercel with `apps/web` as the project root. Root `render.yaml` provisions separate single-instance regional API worlds; each self-registers in `game_servers` from Render's external URL (`PUBLIC_SERVER_URL` is an optional override). All deployments share Postgres and `JWT_SECRET`, but API processes never share simulation memory and must not be autoscaled behind one URL. An isolated draft server uses `CONTENT_CHANNEL=staging`, stays unregistered by default, and is linked from the engine with `NEXT_PUBLIC_STAGING_GAME_URL`. See `docs/DEPLOYMENT.md`.
- **Hideouts ("home base")**: per-user grass-field instance ringed by impassable cliffs (`HIDEOUT_WĂ—HIDEOUT_H`, bed + starter stash). **Players log in at home** and deploy via the door mat â†’ random world spawn (`returnPos` returns you to the safe zone you entered from mid-session). Building is craft-then-place: craft a `kit_*` placeable item, hold it in hand, press `R` to rotate its DB-art ghost in quarter-turns, then click; `c:build {slot,tx,ty,rotation}` consumes the kit and persists the normalized orientation. Base pieces: wood/stone floors are foundations (`FLOOR_TILES` â€” other pieces can be placed on top), walls, doors, fences and lights. Doors swing open when a player crosses them, temporarily allowing enemies, bullets and sight through, then close after players leave. `c:demolish {tx,ty}` in your own camp reclaims the kit (chests must be empty); camp builds are permanent while world builds wear out. Workbench/furnace gate station crafting. Client shows a green tile ghost while placing and a red X marker in demolish mode.
- **Quests**: `quests` table (with `requiresId` unlock chain + `tier`), CRUD via `/api/admin/quests` (editor QUESTS tab), hot-reloaded by the API every 60s; `tools/seed-quests.mjs` seeds the test tree. A job unlocks (and only ticks kills) after its prerequisite is claimed, then remains visible in the HUD tracker; rewards must still be claimed at a trader of its tier. Fetch jobs check inventory at claim.
- **Chat**: ordinary `c:chat` works anywhere, broadcasts within the sender's current instance, and renders as overhead bubbles + log. `/c message` is private relay-wide clan radio and does not create a positional bubble. Guests cannot chat; normal rate limits and admin mutes still apply.
- **Instances**: the server runs named instances â€” `world`, on-demand `h:<userId>` personal hideouts, and `clan:<id>` shared holdouts. Sockets join a room per instance; state broadcasts and ordinary chat are per-instance. Changing instance re-sends `s:init`; `/c` clan radio routes privately to connected clan members across instances on the relay.
- **Clans**: `clans` + `clan_members` + `clan_invites` enforce one clan per survivor and Owner/Officer/Member ranks. `/api/clans` owns management. `clan:<id>` is a 42Ă—30 shared safe hideout persisted in `clans.hideout`; all members build/use storage, Owner/Officers demolish. Entering from home makes the clan exit mat return to the player's personal hideout; entering from a world safe zone returns to that same safe zone, so clan travel cannot bypass extraction. The shared credit treasury uses lease-guarded, transaction-safe transfers and `clan_ledger_entries`. A renewable `clan_hideout_leases` row prevents cross-server last-writer-wins corruption. `StateSnap.mapPlayers` contains server-authorized friend/clan tactical positions only.
- **Extraction**: world maps have extraction beacons (`inst.extracts` â€” authored `extract` objects or `placeExtracts` edge fallback). `E` nearby starts a 5s hold-still action (`EXTRACT_TIME_MS`); success switches you to your hideout with loot intact. Beacons refuse inside safe zones, and `hideoutEnter` no longer warps you to your own camp â€” extraction is the only way home (friends' camps still visitable from safe zones). Death still drops everything.
- **Line of sight**: `losClear()` raycasts blocking tiles/content. `broadcast()` sends bounded per-viewer world snapshots: players, enemies, projectiles, containers and ground loot outside range/LOS are omitted, while population remains aggregate. Enemies hunt last-seen positions and investigate gunshots. The client mirrors vision with an LOS fog overlay. Optional authoritative survivor bots are controlled by published settings and use the same movement/combat/economy rules.
- **High-latency play**: the web client predicts only its local render position using the same terrain, elevation, door, block, stamina and encumbrance rules, then reconciles against `PlayerSnap.ack` without ever sending a position. Aim and cosmetic attack feedback are immediate; damage, ammo, cooldowns and movement remain server-owned. Inputs and 20 Hz state snapshots are volatile/replaceable so congestion drops stale frames instead of replaying them.
- **Stations on floors**: placing a piece on flooring records the floor in `inst.unders`; demolish/destroy/decay restores it. Published player-block definitions and legacy tiles both resolve station behavior. Near-station flags push whenever `lastStationMask` changes, and firepit/furnace fuel is keyed by station tile and persisted for hideouts. Node damage, cooldowns and rerolled variants persist per server/map in `game_world_state`; regrowth weights tree variants (common/ironwood) and rerolls rock-family ore. Friends' camps are visitable from home or safe zones while the owner is online on the same relay; visitors are evicted when owner authority ends.
- **Day/night**: global clock (`DAY_LENGTH_MS`, `isNight()`). At night zombies get `NIGHT_SPEED_MULT`/`NIGHT_AGGRO_MULT`, wolves aggro wider, and one bounded population-scaled surge can spawn outside exposed survivors' sight before clearing at dawn. Client composites an offscreen darkness mask: equipped items with `lightRadius` reveal portable light, torch posts reveal the widest static area, and firepits illuminate only while server-replicated `stationFuel` is positive.
- **Engine visuals/terrain**: `sprites` stores multi-frame RGBA assets; `animations` maps `player` and `mob:<id>` state clips to frame order/timing. Published visuals are sent in `WorldInit` and hot-pushed with `s:visuals`; sheet art remains fallback for non-migrated assets. Every authored map cell has an `AuthoredMap.terrain` ID controlling DB ground art, minimap color, movement/swimming, collision and footsteps. `EngineTerrainDefinition.simulationTile` derives the hidden numeric compatibility grid; there is no admin Base layer. Maps may also carry `elevations` (0-3): one-level grades are hills, jumps over one level block movement/LOS/projectiles as cliffs.
- **Safe zones**: POIs with `safe: true` (trader outposts). No damage in/out, enemies don't aggro, shooting disabled. Hideouts are always safe, and resting there heals (`HOME_REST_HP_PER_S`).
- **Game engine**: `/admin/{slug}` uses `game_content` draft/published documents and immutable `game_content_revisions`. Items/recipes are a versioned runtime catalog sent to clients; other migrated content hot-reloads every 10s. The global admin safety bar can restore content or map history and launch the configured staging raid. Maps load when the world is empty. See `docs/ENGINE.md`.
- **Maps**: server loads the active non-draft row from `maps` or falls back to procedural generation (`apps/api/src/game/mapgen.ts`). Authored maps can attach published resource IDs to tree/rock/ore tiles; those definitions control health, drops, respawn, sprite and sound.
- **Engine audio/animation**: published `sounds` are Web Audio presets referenced by action routes, mobs, resources and animation keyframes. Animation keyframes can override duration and carry event/sound metadata.
- **Dynamic world blocks**: `AuthoredMap.blocks` maps tile indices to published block IDs. Main-world authored blocks are damage-protected regardless of their definition; only player-kit structures registered in `inst.structures` are destructible. The catalog includes authored geometry and player build pieces with pixel, collision, health, drop, sound and `playerPlacement` metadata. Shared `BUILDABLES` tiles remain a station/foundation/persistence compatibility cache.
- **Facing and doors**: player `angle` is always cursor/weapon aim; replicated `facing` follows normalized movement and only turns toward an action while stationary. Clients mirror body art from `facing` while held weapons continue using `angle`. Instances track `openDoors`; nearby players open door blocks, and open doors stop blocking enemy movement, bullets and LOS until they close.

## Conventions

- Socket event names only via `EV` from shared â€” never string literals.
- Never trust client payloads: clamp numbers, bounds-check slots, rate-limit (see `apps/api/src/game/rate-limit.ts` and `docs/ANTICHEAT.md`).
- Env: `apps/api/.env`, `apps/web/.env.local` (gitignored). `JWT_SECRET` must match across both.
- `npm run dev` at root runs everything. `npm run check` must pass; it includes Prisma validation, workspace typechecks, production builds and deterministic gameplay tests.
- `npm run test:load` runs the isolated 100-client websocket benchmark; use `node tools/load-test.mjs --clients=200 --seconds=10` after building to exercise the configured ceiling. Benchmark users, leases, profiles and world state are disposable and removed on exit.
